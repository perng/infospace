在 Infospace 中建立簡報

給作者的使用手冊：從一段想法開始，建立一趟有旁白的 3D 世界導覽。

Infospace 是什麼？

Infospace 以穿越連續 3D 世界的引導式旅程取代投影片。你的內容，像是文字、公式、圖表，會存在於具有氛圍的環境裡的命名位置：傍晚講堂黑板上的粉筆字、漂浮在黑暗虛空中的玻璃板蝕刻公式、化為發光全像圖的資料。觀眾沿著你撰寫的路線從一站移動到下一站，搭配電影式鏡頭運動、選用的語音旁白，以及在世界之間深入切換的傳送門。

身為作者，最重要的概念是：你寫的是意義，框架處理的是空間。你永遠不需要輸入座標、擺放鏡頭或挑選顏色。你只要使用各個世界公布的名稱，說出「這個公式放在中央黑板上，鏡頭靠近到足以閱讀」，layout solver、camera planner 和 skins 就會把它轉成完成後的場景。因為所有視覺形式都從你的語意內容推導而來，同一份文件也會自動產出可閱讀的大綱、字幕與螢幕閱讀器文字。

brief.md              你的想法，一段文字                 （選用，見教學 A）
   │   npm run generate
   ▼
talk.spec.yaml        Authoring Spec：用名稱指定場景與位置，沒有座標
   │   npm run spec:compile
   ▼
document.json         authored document（仍然不含座標）
   ▼   載入時編譯：layout solver → skins → router → camera planner
執行中的 3D 導覽       npm run dev

五分鐘掌握概念

術語意思
World（世界）一個具有自身氛圍與尺度的 3D 環境，例如大理石的 atrium、傍晚的 lecture-hall、黑暗的 math-void。世界已預先建好；你選擇簡報要使用哪些世界。
Station（站位）世界公布的命名內容槽位，例如 board.center 或 plinth.identity。每個站位都有位置與尺寸包絡，但你只會使用它的名稱。每個站位只能放一個場景。
Scene（場景）導覽中的一站：一段內容、它所在的站位、鏡頭如何構圖，以及旁白要說什麼。場景會依照你列出的順序播放。
Skin一段內容的 3D 外觀：chalkboard 會把公式渲染成會自行書寫的粉筆筆畫；hologram 會把長條圖渲染成發光的體積長條。每種內容都有合理的預設 skin。
Camera intent（鏡頭意圖）命名的構圖方式，例如 read-close（正面、可閱讀）或 orbit-focus（適合圖表的四分之三視角）。Planner 會把 intent 和站位尺寸合併，計算實際鏡頭姿態。
Portal（傳送門）世界之間的電影式轉場，鏡頭會鑽入一個世界中的某個物件，並從另一個世界現身，例如直接穿過講堂黑板進入數學虛空。
Branch（分支）現場簡報時可用的選用捷徑，例如「跳到揭曉」。它會顯示給主講者，不會干擾主要路線。
Narration（旁白）每個場景的口述腳本。它以文字儲存，也就是來源真相；音訊片段由文字轉語音工作離線產生，並同步字幕。

快速開始：執行範例

從 repository root 執行：

npm install
cd examples/basel-problem
npm run dev

打開 Vite 印出的本機網址，通常是 http://localhost:5173。你會站在講堂中 「Basel Night Lecture」 的第一塊黑板前，這是一個完整由 AI 生成的講座。按 V 可以聽它自動旁白。

Repo 內附三個範例簡報：

  • examples/svd-tour：旗艦範例，帶你在 atrium 與 spectral world 中理解 Singular Value Decomposition，包含已渲染的動畫。
  • examples/math-primer：最小範例，Euler's identity，從 lecture hall 進入 math void。
  • examples/basel-problem：由 AI pipeline 從一段 brief 生成。教學 A 會沿用它的確切步驟。

Viewer 控制

按鍵動作
→ / Space / PageDown下一站。有 stepwise 動畫的場景會先一次前進一個步驟。
← / PageUp上一站

Minimap 位於底角，會顯示 route graph；presenter console 會顯示目前站點、備註，以及從此處離開的任何 branch 捷徑。

三種建立簡報的方式

路徑你撰寫最適合
A. 從 brief 生成 一段文字（brief.md） 幾分鐘內得到完整初稿，再開始編輯。建議從這裡開始。
B. 手寫 spec 一個 YAML 檔（talk.spec.yaml），每個場景一個區塊 完整掌控每一句話，同時仍然完全不碰座標。
C. 文件層 TypeScript / JSON（authored document） 進階用途：圖片、影片、已渲染的 Manim 動畫、自訂世界。請見 Spec 之外。

路徑 A 和 B 會匯流到同一個檔案：生成流程會產生一份 talk.spec.yaml，之後你再手動編輯。教學 B 的所有內容也都適用於生成出來的 spec。

教學 A：從一段 brief 開始

examples/basel-problem 正是這樣做出來的。你寫一段文字；AI engine 會以框架實際存在的 vocabulary 為依據，所以不會憑空編出不存在的位置，然後草擬完整講座；接著你再檢視並調整。

步驟 1：設定專案

最簡單的起點是複製生成範例並清空它：

cp -r examples/basel-problem examples/my-talk
cd examples/my-talk
rm -rf node_modules dist public/assets/narration talk.spec.yaml talk.provenance.json
# edit package.json: change "name" to "my-talk"
cd ../.. && npm install        # registers the new workspace

還有兩個地方需要個人化：

  • index.html：頁面標題。
  • src/main.tsx：app 要載入哪些 world components。這必須符合你的簡報使用的世界：
<PresentationApp
  journey={journey}
  worlds={{ "lecture-hall": LectureHallWorld, "math-void": MathVoidWorld }}
/>

第一次做簡報時，先保留 lecture-hall + math-void 這一組；參考資料列出全部四個世界。

步驟 2：撰寫 brief

用你自己的段落取代 brief.md。說明講座主題、目標觀眾、大約幾站、故事弧線，以及語氣。以下是實際 Basel brief 的中文譯文，方便感受它的口吻：

# Brief

一場給大二學生的 **Basel problem** 引導式講座，
大約八站，適合做成可自行播放的旁白導覽。

故事：Euler 在 1734 年對 Basel problem 的解法。開場先提出
問題，1 + 1/4 + 1/9 + 1/16 + … 等於什麼？展示這個級數
會收斂，但會以極慢速度爬向極限（partial sums 很適合做成
圖表）。接著帶大家走過 Euler 大膽的一步：把 sin(x)/x 當成
無限多項式，讀出它的 roots，並比較係數來變出答案。
揭曉，也就是 pi squared over six，應該在某個超現實之處成為
戲劇性的高點。結尾說明它為何重要：zeta function 的誕生，
以及通往 Riemann 的道路。

語氣：帶著嚴謹的驚奇感，一場夜間講座，最後抵達黑板無法去的地方。
前半使用 lecture hall 進行鋪陳與級數推導，然後穿過黑板進入 void，
完成 Euler 的躍進與答案揭曉。
好的 brief 有什麼特徵 點出戲劇高點，以及你想讓它在哪裡發生（例如「某個超現實之處」）；提到適合做成圖表的資料；描述語氣。Generator 會把這些線索映射到 worlds、skins 與 camera framings。

步驟 3：生成

npm run generate

這會以你的 brief 和你選擇的世界執行 generator。請看 package.json 裡的 generate script；如果你選了不同世界，就在那裡更改 --worlds lecture-hall,math-void。它會呼叫本機安裝的 AI CLI：預設是 claude CLI，或傳入 --engine codex 使用 Codex CLI。不論哪一個，都必須已安裝並登入。

每份草稿在接受前都會通過完整 compiler 驗證。無效輸出會有一次自動修復，之後仍失敗就會被拒絕。成功執行會寫入三個檔案：

  • talk.spec.yaml：Authoring Spec，以可編輯 YAML 表示的講座。現在這就是你的來源檔。
  • src/journey/document.json：app 載入的 compiled document。
  • talk.provenance.json：brief、engine、model 與嘗試次數的紀錄。

執行 npm run dev，然後走一遍導覽。

步驟 4：檢視與編輯

生成會給你一份結構有效的草稿；判斷仍然是你的工作。走一遍導覽，並批判性地閱讀 talk.spec.yaml。Basel 講座生成後，人工檢視只做了兩個編輯，而且都是 spec 裡的一行：

  • 有個 camera intent 不適合內容，將需要閱讀的公式從 camera: sky-gaze 改成 read-close。
  • 開場段落對黑板來說太長，所以修剪了 body: 文字。

每次編輯後，重新編譯並檢查：

npm run spec:compile
npm run validate
npm run dev

步驟 5：鎖定好的部分，重新生成其餘部分

當一個場景完全正確時，標記它：

  - id: reveal-answer
    locked: true
    title: Pi Squared Over Six
    ...

接著 npm run generate:regen 會根據可能已更新的 brief 重寫講座，同時把所有 locked scenes 逐 byte 保留。這就是編輯迴圈：保留有效的時刻，讓 AI 重做它們之間的連接組織。

教學 B：手寫 spec

Authoring Spec 是一個 YAML 檔：header、一組 scenes，以及選用的 portals 和 branches。你只會使用下面參考表中的名稱，也就是 station names、camera intent names、skin names。

場景剖析

  - id: definition          # kebab-case, unique within the talk
    title: Defining Phi     # shown in the outline, palette, minimap
    world: lecture-hall     # a world template id (listed in `worlds:`)
    station: board.center   # a station that world publishes — one scene per station
    role: detail            # chapter | detail | chart | quote | media | portal
    content:                # what this stop says (see Content kinds)
      kind: formula
      latex: \\varphi = \\frac{1 + \\sqrt{5}}{2}
      spoken: phi equals one plus the square root of five, all over two
    skin: chalkboard        # optional — omit for the content kind's default
    camera: read-close      # optional — a camera intent (default read-close)
    narrate: >              # optional — the spoken script for the narrated tour
      Solving the proportion gives a quadratic equation, and its
      positive root is phi.
    notes: Derive it live if the room wants the algebra.   # presenter notes
    arrive: fly             # optional — how the camera travels here: cut | dolly | fly
    locked: false           # optional — survives --regen verbatim when true

只有 id、title、world、station 和 content 是必要欄位。預設值都很好用：除非你想要特定效果，否則可以省略 skin 和 camera。

完整講座範例：「The Golden Ratio」

這是一個六場景講座，使用與 Basel 範例相同的兩個世界：在講堂黑板上鋪陳，然後透過黑板傳送門進入虛空迎來收束。這個檔案可直接編譯：

id: golden-ratio
title: The Golden Ratio - a short walk
worlds:
  - lecture-hall
  - math-void
scenes:
  - id: opening
    title: An Old Obsession
    world: lecture-hall
    station: board.left
    role: chapter
    content:
      kind: text
      title: The golden ratio
      body: |
        Cut a line so the whole relates to the long part
        as the long part relates to the short.

        That single sentence defines phi.
    skin: chalkboard
    narrate: >
      For more than two thousand years, one proportion has fascinated
      geometers, painters, and architects. Tonight we meet it properly.

  - id: definition
    title: Defining Phi
    world: lecture-hall
    station: board.center
    content:
      kind: formula
      latex: \\varphi = \\frac{1 + \\sqrt{5}}{2} \\approx 1.618
      spoken: phi equals one plus the square root of five, all over
        two, approximately one point six one eight
    narrate: >
      Solving the defining proportion gives a quadratic equation, and
      its positive root is phi.

  - id: fibonacci
    title: Fibonacci Closes In
    world: lecture-hall
    station: board.right
    role: chart
    content:
      kind: chart
      data:
        type: bar
        title: Ratios of consecutive Fibonacci numbers
        series:
          - { label: 3/2,    value: 1.5 }
          - { label: 8/5,    value: 1.6 }
          - { label: 21/13,  value: 1.6154 }
          - { label: 55/34,  value: 1.6176 }
          - { label: 144/89, value: 1.618 }
      summary: Ratios of consecutive Fibonacci numbers settle toward
        one point six one eight.
    camera: orbit-focus
    narrate: >
      The Fibonacci numbers sneak up on phi. Divide each by the one
      before it, and the ratios settle onto the same constant.

  - id: leaving-the-hall
    title: Through the Board
    world: lecture-hall
    station: wall.west
    role: portal
    content:
      kind: text
      title: A self-similar number
      body: |
        Phi is the number that contains itself.

        To see that, we leave the hall.
    skin: cloudText
    camera: wide-establishing
    narrate: >
      What makes phi special is not its decimals but its self-similarity.
      To see it, we step through the board.

  - id: self-similar
    title: Phi Squared
    world: math-void
    station: plinth.identity
    content:
      kind: formula
      latex: \\varphi^{2} = \\varphi + 1
      spoken: phi squared equals phi plus one
    skin: etchedGlass
    camera: quote-gaze
    narrate: >
      Square phi and you get phi back, plus one. No other positive
      number does this. Every famous property of the golden ratio
      unfolds from this single identity.

  - id: closing
    title: Everywhere and Nowhere
    world: math-void
    station: circle.overlook
    content:
      kind: text
      title: Where phi lives
      body: |
        Pentagons. Sunflowers. Continued fractions.

        phi = 1 + 1/(1 + 1/(1 + ...))
    skin: cloudText
    camera: quote-gaze
    narrate: >
      Phi appears wherever growth repeats its own rule. That is where
      we leave it, glowing over the void.

portals:
  - from: leaving-the-hall
    to: self-similar
    label: Step through the board
    dive: board-dive          # a pose of the source world (lecture-hall)
    emerge: void-emerge       # a pose of the target world (math-void)

branches:
  - { from: opening, to: fibonacci, label: Skip to the evidence, kind: branch }

值得注意的事：

  • Scene order 就是路線。導覽會依照 scenes 列表順序拜訪場景；主路徑不需要手動接 edge。
  • 那裡的 portal 是必要的。leaving-the-hall 和 self-similar 是連續場景，但位於不同世界；compiler 不會猜測世界切換方式，而是要求提供有 dive/emerge pose pair 的 portal。
  • Skins 預設跟隨內容種類。definition 場景省略 skin:，所以它的 formula 會取得 chalkboard。Void scenes 則選用 etchedGlass 與 cloudText 來建立氛圍。
  • 世界內文字是純 ASCII。3D 字型沒有希臘字母或下標。文字 body 與 title 請寫 "phi"、"n squared"、"1/n^2"；真正的符號應該放在 latex: 中，會經由完整數學 pipeline 渲染。
  • 每個公式與圖表都有口述形式（spoken: / summary:），outline、captions 與 screen readers 都使用它。

編譯與迭代

npm run spec:compile     # talk.spec.yaml → src/journey/document.json
npm run validate         # full structural + coverage report
npm run outline          # the talk as a readable text outline
npm run dev              # walk it

validate 會報告 resolution summary，例如有多少 cameras 來自 intents、route coverage、narration coverage，並指出任何錯誤與失敗的確切名稱。outline 是不啟動 3D view 就能校對文字的最快方式。

讓文字符合站位大小 講堂黑板約為 3.8 × 2.8 units；大約三個短段落的 text body 可以放得下。如果 3D view 中文字看起來很擠，就刪字，不要和 layout 對抗。較大的站位（circle.overlook、arrival.overlook）可以容納更多內容。

語音旁白

Narration 是以文字撰寫的，也就是每個 scene 的 narrate: 欄位，並離線渲染成音訊。腳本文字是來源真相；音訊是衍生出來且可快取的 asset。

npm run narration            # print the scripts as JSON (proofread pass)
npm run narration:render     # render audio clips with local TTS

Rendering 會在本機使用 Kokoro text-to-speech engine。需要 model files；請見 packages/cli/generate-narration.py。Clips 會與 word timestamps manifest 一起放在 public/assets/narration/，這正是同步 karaoke captions 的來源。Clips 會以 hash 快取：只有腳本變更過的 scenes 會重新渲染。

在 viewer 中，V 會切換旁白導覽：每一站播放 clip、顯示 captions，並在結束時自動前進。請用口語風格撰寫 narrate:，通常兩到四句，數字和符號要寫成文字，例如 "pi squared over six"，不要寫 "π²/6"。

Validation 會知道 asset 是否過期 npm run validate 會在 script 於 clip 渲染後又被修改時提出警告，所以你不會不小心出貨一份旁白內容和文件不一致的簡報。

鏡頭意圖

Intent構圖
read-close正面、足夠靠近，可以舒適閱讀 body text。預設值。
wide-establishing向後退並越過內容看向背後世界，適合抵達與離開
orbit-focus偏軸四分之三視角，讓圖表與 3D 物件更好看

quote-gaze略低於漂浮文字，向上觀看

你選擇 intent；planner 會根據 station 的 size 與 facing 計算實際 camera position。場景的 camera: 也可以命名某個世界的 fixed poses，例如 prism-approach，用於針對該世界自身幾何的構圖。

Skins 會自動縮放到各自的 station。Skin 必須接受內容種類；把 etchedGlass skin 放在 chart 上會造成 validation error。

內容種類（在 spec 中）

Kind欄位備註
texttitle（選用）、body純 ASCII。大約三個短段落可放進一塊黑板。
formulalatex、spoken簡報尺寸的 LaTeX，一行或一小塊 aligned block。spoken 是數學式的口述讀法。
chartdata、summarydata.type 可為 bar（title、選用 unit/logScale、label/value 的 series）或 scatter（title、xLabel、yLabel、x/y/label 的 points）。只適合小型 inline datasets。summary 是一句話的圖表解讀。

圖片、影片與 Manim 動畫需要磁碟上的 asset files，因此會從 document layer 進入，而不是從 spec 進入。

YAML 中的 LaTeX 請把 latex: 寫成未加引號的 plain YAML scalar，反斜線會原樣通過，例如 latex: \\frac{\\pi^{2}}{6}。如果一定要用雙引號，每個反斜線都要加倍："\\\\frac{\\\\pi^{2}}{6}"。不加引號比較簡單。

欄位必要意思

world是列在 worlds: 中的 world id
station是該 world 公布的 station；每個 station 一個 scene
content是見 內容種類

傳送門與分支

portals:                      # REQUIRED between consecutive scenes in different worlds
  - from: leaving-the-hall    # scene id in the source world
    to: self-similar          # scene id in the target world
    label: Step through the board
    dive: board-dive          # pose name from the source world (optional but recommended)
    emerge: void-emerge       # pose name from the target world

branches:                     # optional shortcuts for live presenting
  - from: opening
    to: fibonacci
    label: Skip to the evidence
    kind: branch              # branch (skip ahead) | return (go back)

Portals 是導覽切換世界的唯一方式；dive/emerge poses 會編排穿越動作，例如加速進入黑板、從天空墜出。Branches 不會中斷主路線，它們會以標記選項出現在 presenter console 與 minimap 上。

Spec 之外：document layer

YAML spec 刻意只涵蓋不需要 external assets 的內容，因此簡報永遠可以渲染。若要更多能力，請編輯 authored document 本身，也就是 spec 下方那一層：

  • 圖片與影片：以 plaque 框住，並引用 public/ 中的檔案。
  • Manim 動畫：用 Python 撰寫 Manim scene；render job 會產生帶 cuepoints 的透明 clip，而 projection skin 會把它合成播放在世界上。搭配 reveal: stepwise 時，每按一次 → 會前進一個 animation step，旁白 cue marks 也會在 self-running tour 中驅動同樣步驟。
  • 手動微調構圖：當 intent 不夠時，可使用 explicit camera poses。

examples/svd-tour 展示了這些能力，請看它的 src/journey/project.ts。Document 仍然是 positionless，也就是 stations 與 intents，只是它會使用框架完整 vocabulary。完整 data model 位於 docs/design.md，而 docs/creation-guide.md 是濃縮參考，雖然是為 AI assistants 撰寫，但對想把所有欄位放在同一頁閱讀的人類作者同樣有用。

疑難排解

症狀 / 訊息修正方式
…are consecutive but in different worlds — add a portal… 兩個前後相接的 scenes 在沒有 portal 的情況下切換 worlds。請在 portals: 中加入一筆從第一個到第二個的 entry，並附上 dive/emerge pose pair。
station "…" is double-booked by anchors… 兩個 scenes 使用了同一個 station。把其中一個移到空的 station；上方每個 world 的表格就是選單。
…references unknown station "…" (available: …) 可能是 typo，或 station 屬於另一個 world。錯誤訊息會列出有效名稱。
unknown world template "…" worlds: 必須完全使用 template ids：atrium、spectral、lecture-hall、math-void。
公式渲染為空白或錯誤 檢查 YAML escaping（見 LaTeX 提示），並維持 presentation-sized standard LaTeX（amsmath）；exotic packages 無法使用。
文字溢出黑板 Station 比你的段落小。刪字或移到更大的 station；黑板是 3.8 × 2.8。
3D text 中缺少希臘字母 / 符號 世界內文字只支援 ASCII。請拼出來，例如 "pi"、"n squared"；真正的符號請使用 formula scene。
編輯後旁白聽起來過期 重新執行 npm run narration:render，只有更動過的 scripts 會重新渲染。npm run validate 會列出 stale clips。
npm run generate 無法啟動 engine 預設 engine 是 claude CLI，而且必須已登入；否則請安裝並登入 codex CLI，然後在 generate script 加上 --engine codex。
編輯 talk.spec.yaml 後沒有反映出來 App 載入的是 document.json，不是 YAML。每次 spec 編輯後都要執行 npm run spec:compile。
